# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Reproduce all processed data from scratch
python3 src/merge_census_and_geospatial.py       # census merge + subzone+population GeoJSON
python3 src/validate_and_copy_geospatial.py      # copy/validate electoral boundary GeoJSON

# Sanity-check all processed files
python3 scripts/sanity.py                        # requires geopandas (pip install geopandas)

# Run tests
pytest --cov=src --cov-report=term-missing       # 481 tests

# MCMC ensemble analysis (run in order)
python -m src.analysis.cli assign-actual --year 2020
python -m src.analysis.cli assign-actual --year 2025
python -m src.analysis.cli run-ensemble --run-id <run_id> [--n-steps 10000]
python -m src.analysis.cli diff --run-id <run_id> --year-2020-run-id <id> --year-2025-run-id <id>
```

Key dependencies not in requirements.txt that `scripts/sanity.py` needs: `geopandas`. Install with `pip install geopandas` or `conda install geopandas`.

## Architecture

Two layers: a **data pipeline** (preprocessing) and an **MCMC analysis pipeline** (ensemble).

```
data/raw/  →  src/  →  data/processed/  →  src/analysis/  →  output/
```

**Preprocessing scripts in `src/`:**

1. `merge_census_and_geospatial.py` — merges three Census 2020 CSV files (joined on `Number`, the planning area/subzone name) into `master_population_subzone.csv`, then spatially joins population into URA subzone polygons → `subzone_with_population.geojson`.

2. `validate_and_copy_geospatial.py` — copies raw electoral GeoJSON (2020, 2025) to `data/processed/` with CRS and unique `_feature_id` guaranteed.

**Primary join key**: subzone name, normalized to uppercase. Census `Number` matches URA `SUBZONE_N`. Join mismatches (non-residential or small subzones) leave `pop_total = 0` and `subzone_census_id = ""`.

## Data layout

| Path | Contents |
|------|----------|
| `data/raw/` | Immutable originals — never modify |
| `data/raw/census_2020_subzone/` | Three Census 2020 CSVs |
| `data/raw/electoral_boundaries/` | ELD GeoJSON for 2020 and 2025 |
| `data/raw/ura_subzone/` | URA Master Plan 2019 subzone GeoJSON |
| `data/raw/hdb/` | HDB building footprints + property table |
| `data/raw/polling_community/` | Community KML/JSON polling district data |
| `data/raw/sla_cadastral/` | SLA cadastral parcels |
| `data/processed/` | Pipeline outputs; regenerated by scripts |
| `data/reference/pdfs_to_digitize/` | Polling district PDFs/ZIPs needing manual digitization |
| `output/` | Generated maps and reports |

**All processed GeoJSON**: WGS84 (`urn:ogc:def:crs:OGC:1.3:CRS84`), unique polygon ID (`_feature_id` or `FID`).

**Analysis pipeline in `src/analysis/`** (481 tests):

| Module | Role |
|--------|------|
| `io_layer.py` | Load processed GeoJSON / HDB CSVs |
| `graph_build.py` | Build rook-adjacency subzone graph (332 nodes, 850 edges) |
| `communities.py` | Attach HDB town labels to graph nodes |
| `config.py` | Frozen dataclasses: EnsembleConfig, PathsConfig, RunManifest |
| `seed_plans.py` | Seed partition via GerryChain recursive_tree_part |
| `mcmc/` | GerryChain constraints, acceptance, ReCom chain builder |
| `metrics/` | Population, compactness, splitting metrics + registry |
| `ensemble.py` | Full chain driver → metrics.parquet + assignments.parquet + manifest.json |
| `assign_actual.py` | Map actual 2020/2025 electoral plans to subzone assignments |
| `diff_2020_2025.py` | Percentile-rank actual plans against ensemble distribution |
| `reporting/plots.py` | Distribution histograms with actual plan markers |
| `reporting/tables.py` | Summary CSV + markdown table |
| `cli.py` | CLI entry: run-ensemble, assign-actual, diff |

## Known gaps

- **No polling district polygons as GIS layer** — only community KML and PDFs in `data/reference/pdfs_to_digitize/`. Manual digitization needed.
- **KML files** (community polling, SLA cadastral) may need CRS conversion before use in most GIS tools.
- **HDB / cadastral layers** are large; no automatic join to electoral units in the pipeline.
- **Census–subzone match** is incomplete: some URA subzones (non-residential, small) have no Census row.
